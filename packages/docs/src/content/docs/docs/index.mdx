## Start here

Weave is a harness-agnostic prompt and agent-configuration API. You author `.weave` files once, then adapters translate the normalized config into a concrete harness integration.

- New to Weave: start with [Quickstart](tutorials/quickstart/), then [Install and build](how-to/install-and-build/).
- OpenCode users: follow [Configure the OpenCode plugin](tutorials/opencode-plugin/), then [Run an explicit execution](tutorials/first-explicit-execution/).
- Config authors: use the [DSL syntax reference](reference/dsl/syntax/) and field references for [agents](reference/dsl/agents/), [categories](reference/dsl/categories/), [workflows](reference/dsl/workflows/), and [settings](reference/dsl/settings-and-disables/).
- Integrators: read [Architecture](explanation/architecture/), [Engine and adapter boundary](explanation/engine-adapter-boundary/), and the [OpenCode adapter reference](reference/adapters/opencode/).

## What is Weave?

Weave has four public configuration constructs today:

- Agents declare prompts, model intent, mode, skills, routing metadata, and abstract tool policy.
- Categories generate `shuttle-{category}` specialist agents from glob-like pattern metadata.
- Workflows declare ordered explicit execution steps, completion methods, artifacts, extension points, and workflow-level prompt appends.
- Settings and disables configure log level, runtime journal strictness, and disabled agents/hooks/skills.

Only the OpenCode adapter is implemented today. Claude Code and Pi are product targets, but they are not current public adapter implementations.

## Anatomy of a .weave file

```text
agent reviewer {
  description "Strict reviewer"
  prompt "Review the current diff for correctness and missing tests."
  models ["claude-sonnet-4-5"]
  mode subagent

  tool_policy {
    read allow
    write deny
    execute deny
    network deny
    delegate deny
  }
}

category frontend {
  description "Frontend UI, styling, and accessibility"
  patterns ["src/components/**", "**/*.tsx", "**/*.css"]
  prompt_append "Preserve responsive behavior and accessibility."
}

settings {
  log_level INFO
  runtime {
    journal {
      strict false
    }
  }
}
```

Validate it with Bun:

```bash
bun packages/cli/src/main.ts validate --project
```

`weave run` intentionally refuses to run harness runtimes directly. Use `weave init` to configure a harness, then use the harness-native command surface such as OpenCode's `/weave:start` or `/weave:run` when those surfaces are wired by the adapter.
