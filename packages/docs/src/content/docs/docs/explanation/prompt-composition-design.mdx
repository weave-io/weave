---
title: Prompt Composition Design
description: Why prompt rendering is engine-owned and template context is bounded.
---

Prompt composition is engine-owned because it is a deterministic interpretation of Weave config. Every adapter should receive the same normalized prompt for the same resolved config.

The engine renders prompt sources and appends with Mustache against a bounded context. That context exposes only public, schema-aware fields such as agent identity, category metadata, effective tool policy, and delegation targets.

The bounded context is a security and maintenance boundary. Appends cannot read raw artifact contents, chat history, environment variables, or arbitrary object paths. Unsupported Mustache features such as partials and delimiter changes fail composition instead of creating hidden I/O or execution behavior.

Adapters consume `descriptor.composedPrompt`; they do not re-read prompt files or reimplement template behavior.
