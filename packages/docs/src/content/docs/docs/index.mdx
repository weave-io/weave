## What is Weave?

  <p>Weave is a harness-agnostic prompt and agent-configuration framework. A compiler emits a typed, normalized graph from your intent; per-harness adapters translate that graph into native surfaces. Four constructs carry the whole model:</p>
  <ul style="color:var(--text-2);padding-left:var(--s-5);margin-bottom:var(--s-4)">
    <li><strong style="color:var(--text)">Agents</strong> — six canonical roles plus first-class custom roles, each with <code style="font-family:var(--font-mono)">role</code>, <code style="font-family:var(--font-mono)">inputs</code>, <code style="font-family:var(--font-mono)">delegates_to</code>, <code style="font-family:var(--font-mono)">skills</code>, <code style="font-family:var(--font-mono)">model</code>.</li>
    <li><strong style="color:var(--text)">Categories</strong> — intent tags that route work to roles by capability, not by name.</li>
    <li><strong style="color:var(--text)">Workflows</strong> — directed acyclic graphs of stages; cycles and dead-ends are compile errors.</li>
    <li><strong style="color:var(--text)">Skills</strong> — capability signatures bound to roles at compile time.</li>
  </ul>
  <Callout label="Note">
    <p>Weave is intentionally <strong style="color:var(--text)">not a runtime</strong>. It compiles and normalizes intent; execution belongs to whatever harness an adapter targets. The core never imports adapter code or reads harness config.</p>
  </Callout>
</section>

## Anatomy of a .weave file

  <p>A minimal file defines a few agents and the categories that route to them. The compiler infers the rest.</p>
  <CodeBlock name="intro.weave" id="introCode" code={introCode} />
  <div class="term"><span class="p">$</span> weave build intro.weave --adapter claude-code</div>
  <p>The build emits a normalized intent graph and the native surfaces Claude Code expects — system prompts, a capability manifest, and a routing table. Swap <code style="font-family:var(--font-mono)">--adapter</code> and the same intent compiles for OpenCode, Cursor, or a generic CLI without touching the source.</p>
</section>
