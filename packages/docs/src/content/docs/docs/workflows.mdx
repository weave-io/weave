This route is kept for existing links. Current workflow documentation lives in:

- [Workflow DSL reference](reference/dsl/workflows/) for validated syntax and fields.
- [Workflow extension reference](reference/dsl/workflow-extension/) for `extends`, anchored insertion, and `extend before-plan`.
- [Execution lifecycle reference](reference/execution-lifecycle/) for engine-owned lifecycle semantics.
- [Runtime commands reference](reference/runtime-commands/) for adapter-projected command operations.
- [Workflow execution model](explanation/workflow-execution-model/) for why execution is explicit and user-authorized.

The current DSL uses ordered `step` blocks. It does not support stale DAG syntax, fan-in `+` syntax, `role:`, `routes`, or `weave build`.
