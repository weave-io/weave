This tutorial gets you to a valid Weave project config without installing a harness adapter first.

## Prerequisites

- Bun is installed.
- You are in a Git project where you want project-local Weave config.

## Create the config

Run the initializer non-interactively:

```bash
bun packages/cli/src/main.ts init --scope local --yes
```

This creates `.weave/config.weave` and `.weave/prompts/` unless a config already exists. Use `--force` only when you intentionally want a backup and replacement.

## Add one project agent

Edit `.weave/config.weave` and add:

```text
agent reviewer {
  description "Project reviewer"
  prompt "Review the current change for correctness, tests, and maintainability."
  models ["claude-sonnet-4-5"]
  mode subagent

  tool_policy {
    read allow
    write deny
    execute deny
    network deny
    delegate deny
  }
}
```

## Validate

```bash
bun packages/cli/src/main.ts validate --project
```

Expected output includes counts for agents, categories, workflows, disables, and `log_level`.

## Inspect the result

List all resolved agents, including builtins:

```bash
bun packages/cli/src/main.ts prompt list
```

Render the prompt for your agent:

```bash
bun packages/cli/src/main.ts prompt inspect reviewer
```

## What you learned

- Weave always loads builtin config first, then global config, then project config.
- Project config lives at `.weave/config.weave`.
- Prompt inspection shows the composed output adapters consume.
- Execution is explicit and adapter-native; the CLI is for setup, validation, prompt inspection, and runtime inspection.
