## Workspace Shape

Weave is a Bun workspace. Runtime, package management, testing, and builds use Bun. The package split mirrors the architecture boundary: core parsing and validation, config loading, engine composition, adapters, CLI, and docs.

## Package Responsibilities

| Package | Responsibility |
| --- | --- |
| `@weave/core` | Lexer, parser, AST, Zod schemas, validation, config types, and parse errors. |
| `@weave/config` | Builtin DSL defaults, config discovery, prompt path resolution, merge semantics, and plan state provider integration. |
| `@weave/engine` | Prompt composition, descriptor generation, model intent helpers, skill resolution from adapter context, tool policy, lifecycle semantics, runtime store, command operations, and logging. |
| `@weave/cli` | Argument routing, init, migrate, validate, prompt inspection, runtime inspection, and refusal boundary for `weave run`. |
| `@weave/adapter-opencode` | Implemented OpenCode adapter, plugin entry, descriptor translation, command registration/projection helpers, skill discovery, model resolution, and tool-policy mapping. |
| `@weave/docs` | Public Astro/Starlight documentation site and prototype-derived UI shell. |

## Ownership Rules

| If you are changing | Check |
| --- | --- |
| DSL syntax or schemas | Update core tests and public DSL docs. |
| Config discovery or merge | Keep config docs and compatibility behavior aligned. |
| Engine APIs | Confirm the API accepts explicit adapter context and does not scan harness-owned resources. |
| Adapter code | Keep harness-specific discovery, commands, models, and tool names inside the adapter. |
| CLI commands | Update CLI reference and guide workflows in the same change. |
| Public docs UI | Verify base-path-safe links and static build navigation. |

## Common Commands

```bash
bun install
bun run build
bun run typecheck
bun test
bun run docs:build
bun run --filter '@weave/docs' typecheck
```

## Boundary Examples

Correct engine API shape: an adapter passes available skills, available models, selected model, or runtime context into an engine helper and receives normalized output.

Incorrect engine API shape: engine code scans OpenCode directories, calls OpenCode runtime APIs, or registers OpenCode slash commands directly.

## Current Adapter Implementation

Only `@weave/adapter-opencode` is implemented today. Other adapter packages or product targets should not be described as implemented unless source code and tests prove the integration exists.
