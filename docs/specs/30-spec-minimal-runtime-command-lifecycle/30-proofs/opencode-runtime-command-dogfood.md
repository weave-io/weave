## Overview

This document captures sanitized evidence for all six runtime command operations
defined in Spec 30. Each section shows:

1. **User-invoked input** — the explicit handler call (equivalent to a slash command).
2. **Structured pino log output** — engine lifecycle events emitted during execution.
3. **Final result** — the typed `ProjectionResult<T>` returned to the adapter.

Evidence is produced by invoking `RuntimeCommandProjection` methods directly with
in-memory fixtures. This is the documented equivalent path because native OpenCode
TUI slash delivery is degraded in this slice (see [Degraded Affordances](#degraded-affordances)).

No secrets, API keys, credentials, private prompts, or sensitive local paths appear
in this document. Instance IDs and lease IDs are UUIDs generated by the in-memory
runtime store and carry no sensitive information.

```ts

  workflowName:  "tapestry-execution",
  goal:          "Run workflow directly",
  slug:          "run-workflow-directly",
  ownerId:       "weave:run",
  store,
  workflows:     WORKFLOWS,
  adapter,
```


**Final result**:

```
outcome:             success
workflowInstanceId:  <uuid>
workflowName:        tapestry-execution
message:             [/weave:run] Workflow "tapestry-execution" started — instance <uuid>
                     (3 effect(s) applied)
```

**Separation from plan execution proven**: `planStateProvider.planExists` is **not called**
during `handleRunWorkflow`. The `MockPlanStateProvider.planExistsCalls` array remains empty.
This is asserted in `runtime-command-projection.test.ts §4`:
`expect(planStateProvider.planExistsCalls).toHaveLength(0)`.

**Deliberate non-exposure of named workflow as default**: no session hook, idle hook, or
`session.created` event calls `handleRunWorkflow`. The `plugin.ts` `session.created` hook
materializes agents only — it does not call any command operation. This is asserted in
`plugin.test.ts` (session materialization hook does not call `runWorkflow`).

## 4. Abort / Cancel — `/weave:abort`

**User-invoked input** (equivalent to `/weave:abort <instanceId> <leaseId> cancel`):

```ts
projection.handleAbortExecution({
  workflowInstanceId: instanceId,
  leaseId:            leaseId,
  signal:             "cancel",
  store,
})
```

**Lifecycle state transitions** (structured pino log, sanitized):

```json
{"level":40,"module":"control-operations","workflowInstanceId":"<uuid>","status":"completed","msg":"abort-execution: instance is already in a terminal state"}
```

**Final result** (terminal-state guard triggered — instance completed before abort):

```
outcome:     failure
error.type:  command_not_found
message:     [/weave:abort] Not found: execution "<uuid>" — Workflow instance "<uuid>"
             is already in terminal state "completed"
```

**Terminal-state guard proven**: the engine checks the instance status before calling
`handleUserInterrupt`. A completed instance returns `command_not_found` rather than
silently succeeding or corrupting state. Asserted in `runtime-command-operations.test.ts §4`.

**Degraded affordance**: the native TUI abort button is not yet wired to `abortExecution`.
Users invoke it via plugin tool or script. Documented in `DEGRADED_AFFORDANCES` and
asserted in `runtime-command-projection.test.ts §6`.

**Successful abort path** (from `runtime-command-operations.test.ts §4`):

```
Input:   running instance + matching leaseId + signal: "cancel"
Output:  outcome=success, signal=cancel, effects=[cancel-execution]
```

## 5. Blocked-Step Advancement — `/weave:advance`

**User-invoked input** (equivalent to `/weave:advance <instanceId> <leaseId> plan agent_signal`):

```ts
// Instance pre-created in "blocked" state on step "plan"
projection.handleAdvanceStep({
  workflowInstanceId: instanceId,
  leaseId:            leaseId,
  stepName:           "plan",
  completionSignal:   { outcome: "success", method: "agent_signal" },
  store,
  context: {
    workflowName: "tapestry-execution",
    goal:         "Test advance",
    slug:         "test-advance",
    workflows:    WORKFLOWS,
  },
})
```

**Lifecycle state transitions** (structured pino log, sanitized):

```json
{"level":30,"module":"runtime-command-projection","workflowInstanceId":"<uuid>","leaseId":"<uuid>","stepName":"plan","outcome":"success","msg":"handleAdvanceStep — delegating to engine advanceStep"}
{"level":30,"module":"control-operations","workflowInstanceId":"<uuid>","leaseId":"<uuid>","stepName":"plan","outcome":"success","msg":"advance-step operation started"}
```

**Final result**:

```
outcome:   success
stepName:  plan
effects:   1  (dispatch-agent for next step)
message:   [/weave:advance] Step "plan" advanced in instance <uuid>
           (signal: success, 1 effect(s) applied)
```

**Blocked-step advancement proven**: a manually blocked instance transitions from
`blocked` to the next step via `completeStep`. The `effects` array carries the
`dispatch-agent` effect for the next step. Asserted in `runtime-command-projection.test.ts §7`.

**Degraded affordance**: the native TUI step-advance UI is not yet wired to `advanceStep`.
Users invoke it via plugin tool or script. Documented in `DEGRADED_AFFORDANCES` and
asserted in `runtime-command-projection.test.ts §7`.

## 6. Health Summary — `/weave:health`

**User-invoked input** (equivalent to `/weave:health`):

```ts
const healthReport = buildOpenCodeHealthReport();
projection.handleRuntimeHealth({ healthReport });
```

**Lifecycle state transitions** (structured pino log, sanitized):

```json
{"level":30,"module":"runtime-command-projection","harness":"opencode","ready":true,"msg":"handleRuntimeHealth — delegating to engine runtimeHealth"}
{"level":30,"module":"runtime-health","harness":"opencode","ready":true,"commandEntrypointsSupported":true,"degradedCount":8,"unsupportedCount":0,"msg":"runtime-health operation completed"}
```

**Final result**:

```
outcome:                      degraded
harness:                      opencode
ready:                        true
commandEntrypointsSupported:  true
degradedOperations:           8  (token-usage-reporting + 7 optional missing capabilities)
unsupportedOperations:        0
message:                      [/weave:health] Adapter "opencode" is ready with degraded
                              capabilities — 8 degraded, 0 unsupported
```

**Full capability table**: see [`health-summary.json`](./health-summary.json).

**Health summary proven**: `runtimeHealth` is pure — it derives its result entirely from
the adapter-supplied `AdapterHealthReport`. It never fails (always returns `ok`). The
`degraded` projection outcome reflects the 8 warning-level capabilities (optional missing
capabilities and `token-usage-reporting: unsupported`). Asserted in
`runtime-command-projection.test.ts §8`.

## Degraded Affordances

Two operations are documented as degraded in this slice. The engine operations are fully
implemented and tested; only the native TUI delivery path is not yet wired.

| Command | Reason | Equivalent path |
|---|---|---|
| `/weave:abort` | Native TUI abort button not yet wired to `abortExecution` | Call `RuntimeCommandProjection.handleAbortExecution` via plugin tool or script |
| `/weave:advance` | Native TUI step-advance UI not yet wired to `advanceStep` | Call `RuntimeCommandProjection.handleAdvanceStep` via plugin tool or script |

Source: `DEGRADED_AFFORDANCES` constant in
[`packages/adapters/opencode/src/runtime-command-projection.ts`](../../../../packages/adapters/opencode/src/runtime-command-projection.ts).

## Lifecycle State Transition Summary

| Command | Input | Lifecycle methods called | Final state |
|---|---|---|---|
| `/weave:start` | planName + workflowName | `planExists` → `startExecution` → `dispatchStep` → `projectEffect` × N → `completeStep` × N | `completed` |
| `/weave:run` | workflowName (no plan) | `startExecution` → `dispatchStep` → `projectEffect` × N → `completeStep` × N | `completed` |
| `/weave:status` | workflowInstanceId | `inspectExecution` (read-only) | unchanged |
| `/weave:abort` | instanceId + leaseId + signal | terminal-state guard → `handleUserInterrupt` | `cancelled` or `paused` |
| `/weave:advance` | instanceId + leaseId + stepName + signal | `completeStep` | next step dispatched |
| `/weave:health` | healthReport | none (pure) | `RuntimeHealthData` |

## Security Notes

- All instance IDs and lease IDs shown as `<uuid>` are randomly generated UUIDs with
  no sensitive content.
- No API keys, credentials, private prompts, or sensitive local paths appear in any
  log line or result.
- The `hostname` and `pid` fields present in raw pino output are removed from all
  log excerpts in this document.
- The `buildOpenCodeHealthReport()` helper explicitly excludes credentials and
  harness config contents from the health report (see `runtime-command-projection.ts §8`).
- All six operations require explicit user invocation. No session hook, idle hook, or
  `session.created` event calls any command operation.
