## Documentation Changes

Added a complete `## weave init migrate` section covering:

- **Entry paths**: explicit `weave init migrate` and ordinary-init discovery
- **Scope-aware legacy source paths**: table of `local` and `global` sources
- **Canonical migration destinations**: table of `local` and `global` targets, with cross-link to `docs/config-loading.md#config-discovery`
- **`--install-dir` behavior in migrate mode**: explicit statement that `--install-dir` is ignored, with rationale (starter-config scaffolding vs. canonical discovery paths)
- **Preflight summary**: annotated terminal output for both no-overwrite and overwrite cases
- **Safety behavior**: validation-before-write, overwrite backup, source preservation, provenance comment, JSONC comment stripping
- **`--yes` scripting behavior**: non-interactive commands and non-TTY exit behavior
- **Warning semantics**: example warning output, best-effort partial conversion, exit code `0` on warning-bearing success
- **Supported field conversions**: table of all legacy → current DSL mappings
- **Agent namespace rules**: `agents` as builtin overrides, `custom_agents` collision detection, `categories` as `category` blocks
- **Prompt file translation**: bare-filename safety rule, cross-link to `docs/config-loading.md#prompt-file-resolution`
- **Post-migration flow**: continues into normal harness selection

### `docs/config-loading.md` — two new subsections

**Under `## Config Discovery`** — new `### Migration and canonical destinations` subsection:

- States that `weave init migrate` writes only to the canonical paths (`~/.weave/config.weave`, `<projectRoot>/.weave/config.weave`)
- Explains why `--install-dir` is ignored in migrate mode
- Cross-links to `docs/cli.md#weave-init-migrate`

**Under `## Prompt File Resolution`** — new `### Migration and prompt-file translation` subsection:

- Explains that bare filenames are preserved because they map cleanly to `<scopeRoot>/.weave/prompts/<filename>`
- Explains why paths with directory components are warned and skipped
- Directs users to manually place prompt files after migration
- Cross-links to `docs/cli.md#prompt-file-translation`

## Design Decisions

### 5.3 — `--install-dir` tension documented explicitly

The `--install-dir` flag is a starter-config scaffolding option that allows `weave init` to write to non-standard locations (e.g. `./custom-weave`). Migration must not honor this flag because the config loader only discovers `~/.weave/config.weave` and `<projectRoot>/.weave/config.weave`. Documenting this explicitly prevents future contributors from adding `--install-dir` support to migrate mode without understanding the constraint.

### 5.4 — Smoke notes use `MemoryFileSystem`-equivalent fixture paths

The smoke notes use `/tmp/weave-fixture-*` paths that mirror the fixture paths used in the automated tests (`MemoryFileSystem` with `/project` and `/home/user`). This makes the smoke notes directly comparable to the test evidence without requiring real filesystem operations.

### 5.5 — Lint warnings are pre-existing

The 98 lint warnings reported by `bun run lint` are pre-existing unused-import warnings in the codebase (visible in the lint output: `logDestination` unused import in a test file). They are not introduced by Task 5 and do not affect the quality gate result (no errors, only warnings).
