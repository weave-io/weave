## What Weave Is

Weave is a harness-agnostic prompt and agent-configuration API. You describe agents, categories, workflows, prompts, model preferences, and abstract tool policy in `.weave` files. Weave parses that intent into a normalized configuration; adapters translate it into whatever a concrete harness needs.

The current public implementation is intentionally narrower than the long-term product vision. The OpenCode adapter is implemented today. Claude Code and Pi are future or placeholder targets. The CLI helps you initialize, validate, inspect, and migrate configuration, but it does not launch agent harnesses for you. `weave run` refuses execution and points you back to adapter-native, explicit execution.

## Current Support Boundaries

These boundaries are repeated across the guides because they prevent common setup mistakes:

| Topic | Current behavior |
| --- | --- |
| Adapter support | Only OpenCode is implemented. Other adapter names are future or placeholder targets. |
| OpenCode plugin entry | Use `@weave/adapter-opencode/plugin`. |
| Registered OpenCode commands | The plugin registers `/weave:start` and `/start-work` prompt commands. |
| Other `/weave:*` labels | `/weave:run`, `/weave:status`, `/weave:abort`, `/weave:advance`, and `/weave:health` are projection/helper labels unless an adapter explicitly wires them. |
| CLI execution | `weave run` intentionally refuses to start harness runtimes. |
| Eval support | `weave eval run` supports seven text-only suite families. Runtime-backed evals are not part of the current public surface. |
| Stale syntax | DAG/fan-in snippets, colon-style `role:`, `routes`, and `weave build` are historical or unsupported. |

## A Minimal `.weave` File

```weave
agent reviewer {
  description "Strict code reviewer"
  prompt "Review the current diff for correctness, regressions, and missing tests."
  models ["claude-sonnet-4-5"]
  mode subagent

  tool_policy {
    read allow
    write deny
    execute ask
    network deny
    delegate deny
  }
}

category frontend {
  description "Frontend UI, styling, and accessibility"
  patterns ["src/components/**", "**/*.tsx", "**/*.css"]
  prompt_append "Preserve responsive behavior and accessibility."
}

settings {
  log_level INFO
  runtime {
    journal {
      strict false
    }
  }
}
```

Validate a project config with:

```bash
bun packages/cli/src/main.ts validate --project
```

Expected result: the command exits successfully and prints a summary with agent, category, workflow, disabled, and log-level counts. If you pass `--json`, it emits the effective validated config for inspection or automation.
