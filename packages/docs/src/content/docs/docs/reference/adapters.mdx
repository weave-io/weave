<p class="lede">Adapters translate Weave's normalized configuration into concrete harness behavior. The engine never imports adapter code or reads harness config.</p>

## Why adapters matter

<p>The engine should not own harness resource discovery, selected-model lookup, or concrete runtime registration. Those concerns stay with adapters so the core remains portable.</p>
