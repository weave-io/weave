## Weave in one sentence

<p>Weave lets you describe agent systems in a <strong style="color:var(--text)">harness-agnostic DSL</strong>, then materialize that intent inside a concrete harness through an adapter.</p>

## The main pieces

### `.weave` DSL

<p>The DSL declares agents, categories, workflows, prompts, delegation metadata, model preferences, and settings.</p>

### Core

<p><code>@weave/core</code> owns the lexer, parser, AST, validation schemas, and normalized config types.</p>

### Config

<p><code>@weave/config</code> loads builtin defaults, discovers config files, resolves prompt files, and merges project plus global config.</p>

### Engine

<p><code>@weave/engine</code> turns normalized config into adapter-facing descriptors and policy decisions.</p>

### Adapters

<p>Adapters translate Weave's normalized intent into harness-specific behavior, tooling, and runtime integration.</p>

## Design principles

<div class="deflist">
  <div class="row"><dt>DSL-first</dt><dd>Built-in agents use the same model as user-defined agents.</dd></div>
  <div class="row"><dt>adapter-owned runtime</dt><dd>Harness-specific discovery and runtime integration stay outside the engine.</dd></div>
  <div class="row"><dt>explicit policy</dt><dd>Tool access and delegation intent are modeled as configuration, not hidden glue code.</dd></div>
</div>
