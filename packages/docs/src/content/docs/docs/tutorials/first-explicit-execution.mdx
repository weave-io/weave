Weave workflows do not start from idle events, background session observations, or `weave run`. Execution starts only after an explicit user-authorized adapter action.

## Prerequisites

- You configured the [OpenCode plugin](../opencode-plugin/).
- Your config validates.
- You have or create a plan file under `.weave/plans/` when using plan execution.

## Try the CLI refusal first

Run:

```bash
bun packages/cli/src/main.ts run
```

The command exits non-zero and tells you to launch the harness directly. This is intentional: Weave configures harnesses; harnesses run sessions.

## Use an adapter command surface

In OpenCode, use the adapter command surface exposed by the plugin.

For plan-first execution, prefer:

```text
/weave:start my-plan-name
```

The legacy alias is:

```text
/start-work my-plan-name
```

These are prompt-based OpenCode commands. They inject a command template into the configured `tapestry` agent; they are not the same thing as the lower-level engine lifecycle API.

## Named workflow execution

The OpenCode adapter also has runtime projection code for explicit named workflow execution:

```text
/weave:run quick-fix "Fix the failing validation test"
```

The implementation lives in adapter-owned projection helpers and delegates lifecycle semantics to the engine's `runNamedWorkflow` operation. If a given OpenCode command is not wired in your installed version, use the implemented adapter helper from a plugin tool or script rather than expecting `weave run` to do it.

## Inspect runtime state

From the project root:

```bash
bun packages/cli/src/main.ts runtime status
bun packages/cli/src/main.ts runtime journal --limit 20
```

If `.weave/runtime/weave.db` does not exist, the commands exit successfully with a friendly message and create nothing.

## What happened

- The adapter supplied workflow name, goal, slug, runtime store, and projection behavior.
- The engine acquired a lease, created or updated a workflow instance, and emitted abstract lifecycle effects.
- The adapter applied those effects in OpenCode.

Read more in [Execution lifecycle](../../reference/execution-lifecycle/) and [Workflow execution model](../../explanation/workflow-execution-model/).
