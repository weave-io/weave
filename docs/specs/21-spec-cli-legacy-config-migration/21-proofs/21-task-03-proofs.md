## Summary

Task 3 implements best-effort JSONC-to-DSL conversion for the supported subset of legacy top-level fields. The key behaviors implemented are:

1. **`convertLegacyJsonc()`** — exported pure function that strips JSONC comments, parses JSON, and converts each top-level key. Returns `{ dsl: string, warnings: ConversionWarning[] }`. Never throws.
2. **Best-effort partial success** — supported fields are converted and included in the output DSL even when unsupported sections are present. The file is always written when supported content is available.
3. **Supported field mappings**:
   - `disabled_agents` → `disable agents [...]`
   - `disabled_hooks` → `disable hooks [...]`
   - `disabled_skills` → `disable skills [...]`
   - `log_level` → `settings { log_level <VALUE> }` (normalized to uppercase; validated against `LogLevelSchema`)
4. **Unsupported sections** (`workflows`, `continuation`, `analytics`, `background`) — warned and skipped with explicit per-field reasons.
5. **Warning-bearing migrations exit `0`** — `renderMigrateSuccess()` appends the warning summary after the success block; the exit code is unaffected.
6. **Accurate preflight count** — `runMigrateMode()` now pre-converts the source before showing the preflight summary so `skippedWarningCount` reflects the real conversion result.
7. **JSONC comment stripping** — `//` line comments and `/* */` block comments are stripped before `JSON.parse` so JSONC source files parse correctly.

## Implementation Notes

- `convertLegacyJsonc()` is exported from `packages/cli/src/commands/init.ts` as a named export alongside `ConversionResult` and `ConversionWarning` types. This keeps the conversion logic directly testable without going through the full `runInit()` flow.
- `performMigrationWrite()` now calls `convertLegacyJsonc(sourceContent)` and passes the result to `buildMigratedContent()`. The return type is extended to `{ backedUp: boolean; warnings: ConversionWarning[] }`.
- `buildMigratedContent()` uses `conversion.dsl` when non-empty; falls back to `starterConfig()` when the source produces no convertible content (e.g. empty `{}`). This preserves the Task 2 behavior for empty/unparseable sources.
- `renderMigrateSuccess()` accepts an optional `warnings` field and appends `renderConversionWarnings()` when warnings are present. The success block and exit code are unaffected.
- `runMigrateMode()` now pre-converts the source before building the final `MigrationPlan` so `skippedWarningCount` in the preflight summary reflects the actual conversion result.
- The JSONC comment stripper is a minimal regex-based implementation sufficient for the common cases in `weave-opencode.jsonc` files. Edge cases (comments inside string literals) are acceptable for migration tooling.
- All Task 2 tests (39 tests) continue to pass without modification — the `performMigrationWrite` signature change is backward-compatible at the call sites.
