Use [OpenCode](opencode/) for current adapter behavior.

## Adapter responsibilities

Adapters own harness resource discovery, available-model lookup, selected-model lookup, concrete tool names, plugin/config generation, command labels, and effect projection.

The engine owns normalized descriptors, prompt composition, abstract policy, model intent helpers, lifecycle semantics, and runtime state.
