The CLI is a setup and inspection tool. It does not launch harness runtimes.

## Global flags

| Flag | Meaning |
| --- | --- |
| `--help`, `-h` | Show command help. |
| `--version`, `-V` | Show version when no command is provided. |
| `--json` | Emit JSON for commands that support it. |
| `--yes`, `-y` | Non-interactive confirmation for init and migrate flows. |
| `--force` | Allow overwrites where supported; migration and init create backups as implemented. |

## `weave init`

Create config and optionally install detected harness integration.

```bash
weave init
weave init --scope local --yes
weave init --scope global --yes
weave init --scope local --install-dir .weave --yes
weave init --scope local --harness opencode --yes
weave init --scope local --all-harnesses --yes
```

Flags:

| Flag | Meaning |
| --- | --- |
| `--scope global|local` | Choose `~/.weave` or project `.weave`. |
| `--install-dir <dir>` | Override the directory written by init. |
| `--harness <name>` | Configure one detected harness. |
| `--all-harnesses` | Configure all detected supported harnesses. |

## `weave init migrate`

Migrate legacy OpenCode JSONC config into canonical `.weave` DSL.

```bash
weave init migrate --scope local
weave init migrate --scope global
weave init migrate --scope local --yes
weave init migrate --scope local --yes --force
```

Local source is `./.opencode/weave-opencode.jsonc`; global source is `~/.config/opencode/weave-opencode.jsonc`. Destinations are `.weave/config.weave` and `~/.weave/config.weave`.

## `weave validate`

Validate config.

```bash
weave validate
weave validate --project
weave validate --global
weave validate --path .weave/config.weave
weave validate --path .weave/config.weave --json
```

Without `--path`, `--project`, or `--global`, validation loads the effective config from builtins, global config, and project config.

## `weave prompt`

Inspect composed prompts.

```bash
weave prompt list
weave prompt list --json
weave prompt inspect loom
weave prompt inspect loom --json
```

## `weave runtime`

Inspect the default runtime store without mutating it.

```bash
weave runtime status
weave runtime journal
weave runtime journal --limit 20
```

The default database path is `.weave/runtime/weave.db` relative to the current project.

## `weave run`

`weave run` intentionally refuses to run harness runtimes directly and exits with guidance to launch the harness yourself, such as `opencode`. Use adapter-native explicit execution commands inside the harness.
